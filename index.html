# MEF - Make Education Fashionable A web application for **MEF (Make Education Fashionable)** — empowering learners and educators through quality education and accessible, modern digital experiences. **Repository:** [https://github.com/ICEP-DEV/MEF.Frontend](https://github.com/ICEP-DEV/MEF.Frontend) --- ## Table of Contents - [Overview](#overview) - [Tech Stack](#tech-stack) - [Project Structure](#project-structure) - [Prerequisites](#prerequisites) - [Cloning the Repository](#cloning-the-repository) - [Installation](#installation) - [Running the Application](#running-the-application) - [Available Scripts](#available-scripts) - [Styling](#styling) - [Project Conventions](#project-conventions) --- ## Overview This repository contains the **MEF** frontend: a React-based app that provides a welcoming homepage with a distinctive curved marquee component. The structure supports adding further pages and navigation (e.g. About, Students, Enquiries) as the project grows. --- ## Tech Stack | Layer | Technology | |-------------|------------| | **UI** | React 19 | | **Build** | Vite 7 | | **Styling** | TailwindCSS 4 + custom CSS | | **Routing** | React Router DOM 7 | - **React 19** – UI library - **Vite 7** – Build tool and dev server - **TailwindCSS 4** – Utility-first CSS (via `@tailwindcss/vite`) - **React Router DOM 7** – Client-side routing - **CSS** – Custom styles in `src/styles/` --- ## Project Structure The repository is organized for clarity and maintainability. ``` MEF.Frontend/ # Repo root (this folder) ├── public/ # Static assets (images, favicon, etc.) │ └── vite.svg ├── src/ │ ├── components/ # Reusable UI components │ │ └── CurvedLoop.jsx │ ├── pages/ # Route-level / page components │ │ └── HomePage.jsx │ ├── styles/ # Stylesheets │ │ ├── custom.css │ │ └── homepage.css │ ├── lib/ # Utilities, helpers, API clients │ │ └── index.js │ ├── constants/ # Routes, config, copy │ │ ├── routes.js │ │ └── site.js │ ├── App.jsx # Root app + router setup │ ├── main.jsx # Entry point │ └── index.css # Global CSS (Tailwind + custom imports) ├── index.html # HTML shell ├── vite.config.js # Vite configuration ├── package.json # Dependencies and scripts ├── package-lock.json ├── .gitignore └── README.md # This file ``` ### What belongs in each folder | Folder | Purpose | Examples of what to put here | |--------|---------|------------------------------| | **`public/`** | Static files served as-is at the root URL. | `logo.jpeg`, `vite.svg`, favicons, fonts, PDFs. No global installs are required. --- ## Running the Application ### Development (with hot reload) ```bash npm run dev ``` Vite starts the dev server (typically `http://localhost:5173`). Open that URL in your browser; changes hot-reload. ### Production build and preview ```bash npm run build ``` Output is in `dist/`. To preview the production build locally: ```bash npm run preview ``` --- ## Available Scripts | Command | Description | |-------------------|--------------------------------------| | `npm run dev` | Start development server (Vite HMR) | | `npm run build` | Build for production | | `npm run preview` | Serve the production build locally | --- ## Styling - **Tailwind** — Use utility classes in JSX (e.g. `className="flex items-center p-4"`). Configured via `@tailwindcss/vite` in `vite.config.js`. - **Custom CSS** — Use `src/styles/custom.css` for global overrides and `src/styles/homepage.css` (or new files) for page-specific styles. --- ## Project Conventions - **Naming:** PascalCase for React components and their files (e.g. `HomePage.jsx`, `CurvedLoop.jsx`). camelCase for utilities and constants in `lib/` and `constants/`. - **Styles:** Prefer Tailwind for layout and utilities; use `styles/` for custom or page-specific CSS. - **Routing:** Routes are defined in `App.jsx`; add new pages under `src/pages/` and register them in ``. - **Constants:** Use `src/constants/routes.js` and `src/constants/site.js` for paths and copy instead of hardcoding in components. - **Assets:** Place images and other static files in `public/` and reference them with absolute paths (e.g. `/logo.jpeg`).
